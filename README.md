# Frontend TypeScript Sandbox

This repository contains a collection of mini front-end exercises completed as part of the ALX **Frontend JavaScript** program. Each task explores a specific concept or tooling setup that is common in modern TypeScript projects.

## Current Task: `0x04-TypeScript/task_0`

The first task focuses on setting up a minimal TypeScript + Webpack pipeline and rendering a simple table of students in the browser.

Key files:

- `js/main.ts` – defines a `Student` interface, creates sample data, and renders it into an HTML table.
- `webpack.config.js` – configures Webpack, TypeScript transpilation via `ts-loader`, source maps, and dev tooling plugins.
- `tsconfig.json` – compiler options (ES5 target, DOM libs, strict-ish defaults).
- `eslintrc.js` – linting rules powered by `@typescript-eslint`.

## Getting Started

1. Install dependencies

   ```bash
   npm install
   ```

2. Launch the development server (auto-opens a browser tab)

   ```bash
   npm run start-dev
   ```

3. Build the production bundle

   ```bash
   npm run build
   ```

4. Run unit tests (Jest + ts-jest)
   ```bash
   npm test
   ```

## Development Notes

- Webpack dev server uses inline source maps for easier debugging.
- `ForkTsCheckerWebpackPlugin` ensures type-checking runs in a separate process.
- The HTML template is auto-generated by `HtmlWebpackPlugin`; the rendered table comes from `main.ts`.
- When adding new TypeScript files, remember to update imports in `main.ts` or adjust the Webpack entry point.
